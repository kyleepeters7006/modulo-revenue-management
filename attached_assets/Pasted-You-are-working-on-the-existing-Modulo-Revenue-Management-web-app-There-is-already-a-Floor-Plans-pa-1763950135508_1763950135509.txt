) while preserving the current styling and navigation.

Left panel: Interactive campus map + floor selector

Right panel: Filtered unit list

The behavior should mirror SightMap:

Filters (top bar)

Reuse existing filters where possible.

Bedrooms (Any, Studio, 1, 2, 3+)

Floor Plan name (Any, specific plan names)

Square Footage (Any, range buckets)

Care Level (Any, IL / AL / MC / HC or whatever matches our data)

Add Reset button to clear all filters.

Add a search box for unit number or floor plan name.

Floor selector

Vertical control to choose floor (e.g., 1–5) if our data includes floors.

Clicking a floor:

Changes which units are drawn on the map overlay.

Updates the unit list and “matches” count.

Campus map + unit overlay

Use our existing campus/floor images if available; otherwise create placeholder images and clearly comment where they should be replaced.

On top of the background, render units as clickable shapes (SVG polygons or positioned overlays) based on our current data model.

Each unit should have:

A visual state for:

Default

Hover

Selected

Color based on care level (e.g., distinct colors for IL / AL / MC / HC).

Hover: show a small tooltip with unit #, beds/baths, sq ft, and price.

Click:

Select that unit.

Highlight it on the map.

Scroll and highlight the corresponding unit row in the list.

Unit list (right side)

Show “X Matches – Floor Y” at the top.

Below, a scrollable list of units that match the current filters + floor.

Each row should show at least:

Unit #

Beds / baths

Sq ft

Care level

Price / rent and availability date if present.

Clicking a row:

Selects and highlights the corresponding unit on the map.

Unit detail card

Unit number and floor plan name

Beds / baths

Sq ft

Price and any specials

Care level

Availability date

“View pricing history”
